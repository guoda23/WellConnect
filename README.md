# WellConnect

WellConnect is a computational modelling project that simulates how social connections form in group-based urban mental health interventions. Under the hood, it estimates pairwise connections between group members based on similarity between individual traits (homophily), then simulates how depressive symptoms may change or spread within the resulting social network.

The current repository is organized around scientific experiments, specifically:

- Retrievability of homophily functions from simulated group connections.
- Depression transmission dynamics under different group compositions and homophily assumptions.

## What This Repository Contains

- Core orchestration logic in `src/WellConnectController.py`.
- Group creation strategies in `src/models/group_creation_strategies/`.
- Homophily functions in `src/models/homophily_functions.py`.
- Transmission models in `src/models/transmission_models/`.
- Experiment runner scripts in `Scripts/experiment_scripts/`.
- Result and plotting scripts in `Scripts/result_scripts/`.
- Configuration files in `Config_files/`.
- Generated experiment outputs in `Experiments/`.
- Generated figures and processed outputs in `Results/`.
- Synthetic datasets, demographic data, and LLM-generated data in `Synthetic data/` (includes notebooks for statistical analysis).

## Main Workflows

### 1) Homophily Function Retrievability

Goal: test whether original trait weights can be recovered from simulated connection structures.

- Deterministic runs:
	- Script: `Scripts/experiment_scripts/homophily_f_retrieval/run_experiments_deterministic_homophily_function_retrievability.py`
	- Config: `Config_files/homophily_f_retrieval/config_deterministic_homophily_f_retrievability.json`
- Stochastic runs:
	- Script: `Scripts/experiment_scripts/homophily_f_retrieval/run_experiments_stochastic_homophily_function_retrievability.py`
	- Config: `Config_files/homophily_f_retrieval/config_stochastic_homophily_f_retrievability.json`

### 2) Transmission Experiments

Goal: simulate depression dynamics in groups under varying homophily structure, entropy, and noise.

- Script: `Scripts/experiment_scripts/transmission/run_experiments_transmission.py`
- Config: `Config_files/transmission/config_transmission.json`

## Setup

This repository includes a `requirements.txt` file with the Python packages used by the project.

Create and activate a virtual environment:

```bash
macOS/Linux:
python3 -m venv .venv
source .venv/bin/activate

Windows (PowerShell):
py -3 -m venv .venv
.venv\Scripts\Activate.ps1

Install dependencies:
pip install -r requirements.txt
```

Notes:

- Some scripts may work with a subset of these packages, depending on what you run.
- You run scripts from the repository root so internal imports resolve consistently.

## How To Run

From repository root:

### Run Deterministic Retrievability Experiments

```bash
python Scripts/experiment_scripts/homophily_f_retrieval/run_experiments_deterministic_homophily_function_retrievability.py
```

### Run Stochastic Retrievability Experiments

```bash
python Scripts/experiment_scripts/homophily_f_retrieval/run_experiments_stochastic_homophily_function_retrievability.py
```

### Run Transmission Experiments

```bash
python Scripts/experiment_scripts/transmission/run_experiments_transmission.py
```

## Analyze Results

The result scripts are currently configured with explicit batch folder paths inside each file. Edit the batch path variables to point to your latest run, then execute:

```bash
python Scripts/result_scripts/run_deterministic_results.py
python Scripts/result_scripts/run_stochastic_results.py
python Scripts/result_scripts/run_transmission_results.py
```

Outputs are written under `Results/`.

## Output Structure

Experiment scripts create timestamped batches for reproducibility, for example:

- `Experiments/homophily_function_retrievability/deterministic/batch_<timestamp>/`
- `Experiments/homophily_function_retrievability/stochastic/batch_<timestamp>/`
- `Experiments/transmission/batch_<timestamp>/`

Each batch includes a copied `config_used.json`, and per-seed/per-run subfolders with serialized experiment artifacts.

## Reproducibility Notes

- Seeds are controlled through config files (`seeds`, `seeds_group_formation`, `seeds_transmission`).
- Configs are copied into each batch folder at runtime.
- Keep a record of the exact script + config combination for each figure reported in manuscripts.

## Current Scope and Future Adaptation
